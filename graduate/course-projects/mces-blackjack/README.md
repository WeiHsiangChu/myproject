# ♠️ Blackjack – Monte Carlo Exploring Starts (MCES)

This project implements **Monte Carlo Exploring Starts (MCES)** to solve the *Blackjack* reinforcement learning problem, reproducing Figure 5.2 from Sutton & Barto's *Reinforcement Learning: An Introduction*.  
The goal is to evaluate the optimal policy (π\*) and value function (V\*) under different rules for the dealer’s behavior on **soft 17**.

📓 [View Notebook](chu_homework4.ipynb)  
📄 [View Report](chu_homework4.pdf)

---

## 🧠 Problem Overview

- **Blackjack Rules**
  - Player starts with two cards; may **hit** or **stick** (must stick at total ≥ 12).
  - Dealer reveals one card and follows a **soft-17 rule** (two variants below).
  - Ace is **usable (11)** if it doesn’t cause bust; otherwise counts as **1**.
  - **Rewards**
    - Player bust → `-1`
    - Dealer bust → `+1`
    - Player > Dealer → `+1`
    - Player < Dealer → `-1`
    - Draw → `0`

- **Soft-17 Cases**
  1. Dealer **hits** on soft 17 (Ace+6).  
  2. Dealer **sticks** on soft 17 (Ace+6).  
     - Dealer must stick on *soft 18* (Ace+7), *soft 19* (Ace+8), and *soft 20* (Ace+9).

- **Objective**  
  Learn the **optimal policy (π\*)** and **value function (V\*)** with MCES.

---

## 🛠 Implementation Details

- **Algorithm:** Monte Carlo Exploring Starts (first-visit)
- **Episodes:** 500,000
- **Behavior:** Random exploring starts; rewards assigned at the end of each episode
- **Updates:** Maintain returns for first visits; update `Q(s,a)` from averaged returns
- **Policy Improvement:** Greedy w.r.t. `Q(s,a)` after each update
- **Outputs:** π\* decision maps and V\* surfaces for both soft-17 cases, each with/without a usable Ace
